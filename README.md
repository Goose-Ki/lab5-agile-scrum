# Лабораторная работа №5 — Agile и Scrum

## Football Match Tracker

Football Match Tracker — учебный проект, моделирующий процесс командной разработки приложения для отслеживания футбольных матчей.
Приложение позволяет просматривать футбольные команды, матчи и игроков, а также обновлять счёт матчей.
Основная цель лабораторной работы — изучение Agile-подхода и Scrum-процесса на практике.
Все требования к системе были оформлены в виде product backlog с использованием GitHub Issues.
Для управления задачами использовались labels, отражающие статус задач, приоритет и сложность (story points).

---

## Product Backlog

Product backlog проекта представлен в виде GitHub Issues.
Каждый issue соответствует отдельной функциональной или нефункциональной задаче и содержит описание и критерии готовности (Definition of Done).

---

## Sprint Planning

Для моделирования Scrum-процесса был спланирован один спринт — **Sprint 1**.
В спринт были отобраны 5 задач из product backlog.
Для задач спринта были назначены следующие атрибуты:
- label  — принадлежность к текущему спринту;
- labels  и  — статус выполнения задачи;
- labels приоритета ();
- оценка сложности в story points.

---

## Sprint Review

В рамках Sprint 1 были выполнены и закрыты следующие задачи:
- Добавление футбольного матча.
- Обновление счёта матча.

В ходе спринта часть задач находилась в статусе , а одна задача осталась незавершённой и была перенесена в следующий спринт.

По итогам спринта был реализован базовый функционал для работы с футбольными матчами.

---

## Sprint Retrospective

**Что прошло хорошо:**
- Product backlog был корректно сформирован в виде GitHub Issues.
- Использование labels позволило удобно отслеживать прогресс задач.
- Scrum-процесс был успешно смоделирован в рамках одного спринта.

**Что пошло не так:**
- Некоторые задачи оказались сложнее, чем ожидалось.
- Оценка сложности задач не всегда точно отражала фактические трудозатраты.

**Что можно улучшить в следующем спринте:**
- Более точно оценивать задачи в story points.
- Детальнее прорабатывать задачи перед добавлением в спринт.
- Улучшить планирование спринта с учётом рисков.
